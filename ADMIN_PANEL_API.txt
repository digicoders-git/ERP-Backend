ADMIN PANEL OPTIMIZED API ENDPOINTS
===================================

BASE URL: http://localhost:5000/api/admin-panel

AUTHENTICATION: All endpoints require JWT token in Authorization header
Format: Authorization: Bearer <token>

=== STAFF MANAGEMENT ===

GET /staff/all
- Get all staff members with pagination
- Query: page=1, limit=20, search='', status='all'|'active'|'inactive'
- Response: { success, data: [], pagination }
- Load Time: <1 second

GET /staff/stats
- Get staff statistics (total, active, inactive, avg salary, by qualification)
- Response: { success, data: { total, active, inactive, avgSalary, byQualification } }
- Load Time: <500ms

GET /staff/with-attendance
- Get staff with today's attendance status
- Query: branch, limit=10
- Response: { success, data: [] }
- Load Time: <800ms

GET /staff/leaves
- Get staff leave requests
- Query: branch, status='all'|'pending'|'approved'|'rejected'
- Response: { success, data: [] }
- Load Time: <1 second

GET /staff/dashboard
- Get staff dashboard overview
- Response: { success, data: { stats, recentStaff, onLeave, todayAttendance } }
- Load Time: <1 second

POST /staff/create
- Create new staff member
- Body: { name, email, mobile, qualification, experience, salary, address }
- Response: { success, message, data }

PUT /staff/update/:staffId
- Update staff member
- Body: { name, email, mobile, qualification, experience, salary, address, status }
- Response: { success, message, data }

DELETE /staff/delete/:staffId
- Delete staff member
- Response: { success, message }

=== PERFORMANCE METRICS ===

All endpoints optimized for:
- Database indexing on frequently queried fields
- Aggregation pipelines for complex queries
- Lean queries for read-only operations
- Parallel Promise.all() for concurrent queries
- Pagination for large datasets
- Caching for repeated requests

Expected Load Times:
- Stats endpoints: <500ms
- List endpoints: <1 second
- Dashboard endpoints: <1 second
- Search endpoints: <1.5 seconds

=== ERROR RESPONSES ===

All errors follow this format:
{
  "success": false,
  "message": "Error message",
  "error": "Detailed error",
  "timestamp": "2024-01-01T00:00:00.000Z"
}

Status Codes:
- 200: Success
- 201: Created
- 400: Bad Request
- 401: Unauthorized
- 403: Forbidden
- 404: Not Found
- 500: Server Error
